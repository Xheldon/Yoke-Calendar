快速上手

下载对应平台的安装包并安装。首次启动是一个空文档（只含默认阶段），你可以：

  • 点左上角 新建需求（快捷键 ⌘N / Ctrl+N）创建一个需求；
  • 或直接在右侧日历上 拖拽划选 一段时间，开始排期。

排期操作

划选排期

在日历上按住拖动划选一段日期，弹出排期窗口。三种模式自动判断：

  • 新建：没有选中需求时——填需求名、选业务线、选阶段；
  • 排期：选中了某个未排满的需求时——为它的某个阶段排期；
  • 调整：双击日历上已有的阶段色条——只调整它的排除天。

排除天

排期窗口里可勾选 排除天（周末、节假日默认可一键排除），被排除的日期不计入工作日。

拖拽微调

落库后，直接在日历上拖动色条 移动 / 缩放 即可微调日期，所见即所得。

业务线 / 阶段 / 标签

  • 业务线：给需求分组的维度（用颜色区分），在侧栏 ⋯ → 业务线管理 维护；
  • 阶段：需求的推进阶段（默认 开发 / 联调 / 测试 / 上线），可在 阶段管理 自定义增删；
  • 标签：跨业务线的标记（如「人」），用于筛选。
这是一个排期记录工具，不是任务管理——排期本身不分「完成 / 未完成」，只有 过去 / 进行中 / 未来。最后一个已排阶段结束日早于今天，即归入「已结束」历史区。某些阶段不排就表示不关心，不影响判断。

多端同步

设置 → 同步，二选一：

  • GitHub：登录后指定一个私有仓库（owner/name），数据以 yoke.json 存入；
  • WebDAV：填入坚果云 / Nextcloud 等的 WebDAV 地址与应用密码。

自动同步：可设 30 分钟 / 1 / 2 / 6 小时。后台定时对比本地与云端，以「更新时间较新」的一方为准（覆盖前自动留本地备份）。密钥经系统加密保存在本机，不随数据上传。

AI 总结

设置 → AI 功能，两种调用方式：

  • API 接口：内置 OpenAI / Anthropic / Gemini / DeepSeek / 通义 / Kimi / 智谱 / 豆包 / OpenRouter / Groq / xAI / Mistral / Ollama 等预设——选服务商、填 Key 即可；每个服务商各自记住自己的 Key 与模型，切换不串。
  • 本地 Agent：复用本机已登录的 Claude Code / Codex CLI，免 API Key。

在 AI 窗口选时间范围 + 提示词（内置周报 / 风险盘点 / 对上汇报），一键生成总结。

日历订阅 / 节假日

  • 订阅：设置 → 日历订阅，添加任意 .ics 地址或导入文件，作为只读叠加显示；
  • 节假日：内置中国法定节假日 + 农历 + 调休，可在 设置 → 日历 切换地区。

数据与备份

  • 本地优先：数据存在你本机的数据目录，离线完全可用；
  • 数据目录：设置 → 系统 可查看 / 更改（「保险库」）；
  • 自动备份：每次保存前自动滚动备份，可在 设置 → 系统 → 备份 一键恢复。

自动更新

应用启动后会在后台静默检查并下载更新，左下角出现进度小窗；下载完成后点 重启更新 即生效（不点也会在下次退出时自动安装）。也可在 设置 → 关于 手动「检查更新」。安装包经签名 + 公证。

常见问题

Windows 安装能选路径吗？

能。安装向导带「选择安装目录」页，默认按用户安装（免管理员），可改到任意盘。

macOS 提示「无法验证开发者」？

正式发布的包已签名 + 公证，正常不会提示。若遇到，右键 → 打开 即可。

数据会上传到云吗？

默认不会。只有你显式开启同步后，数据才会写入你指定的 GitHub 私有仓库 / WebDAV；密钥永远只在本机。

支持哪些平台 / 架构？

macOS（Intel + Apple Silicon）、Windows（x64 / ARM64 / x86）、Linux（AppImage）。

Getting started

Download the installer for your platform and install it. The first launch is an empty document (with just the default phases). You can:

  • click New requirement in the top-left (shortcut ⌘N / Ctrl+N) to create one;
  • or drag to select a span of time right on the calendar to start scheduling.

Scheduling

Drag to schedule

Press and drag across a date range on the calendar to open the scheduling dialog. It picks one of three modes automatically:

  • New — when no requirement is selected: enter a name, pick a business line, pick a phase;
  • Schedule — when a not-yet-complete requirement is selected: schedule one of its phases;
  • Adjust — double-click an existing phase bar on the calendar: edit only its excluded days.

Excluded days

In the dialog you can tick excluded days (weekends and holidays can be excluded in one click); excluded dates don't count as working days.

Drag to fine-tune

Once saved, drag the color bar on the calendar to move / resize it — what you see is what you get.

Lines / phases / tags

  • Business lines — the dimension that groups requirements (distinguished by color); manage them under sidebar ⋯ → Manage business lines;
  • Phases — a requirement's stages (dev / integration / QA / release by default); add or remove your own under Manage phases;
  • Tags — cross-line markers (e.g. a person) used for filtering.
This is a scheduling record tool, not task management — a schedule itself has no "done / not done", only past / in progress / future. When the last scheduled phase ends before today, the requirement moves into the "Ended" history area. Leaving some phases unscheduled just means you aren't tracking them — it doesn't affect this.

Multi-device sync

Settings → Sync, pick one:

  • GitHub — after signing in, point to a private repo (owner/name); data is stored as yoke.json;
  • WebDAV — enter the WebDAV URL and app password for Nextcloud, ownCloud, etc.

Auto-sync — every 30 min / 1 / 2 / 6 hours. It periodically compares local and cloud and keeps whichever was updated more recently (a local backup is kept before any overwrite). Keys are encrypted by the system on your machine and never uploaded with your data.

AI summaries

Settings → AI, two ways to call it:

  • API — presets for OpenAI / Anthropic / Gemini / DeepSeek / Qwen / Kimi / Zhipu / Doubao / OpenRouter / Groq / xAI / Mistral / Ollama and more — pick a provider, enter a key. Each provider remembers its own key and model, so switching never crosses wires.
  • Local agent — reuse the Claude Code / Codex CLI you're already signed into locally, no API key needed.

In the AI window, choose a time range + prompt (built-in weekly report / risk review / status update) and generate a summary in one click.

Calendar feeds / holidays

  • Feeds — Settings → Calendar feeds, add any .ics URL or import a file to show as a read-only overlay;
  • Holidays — built-in Chinese statutory holidays + lunar calendar + make-up workdays; switch region under Settings → Calendar.

Data & backup

  • Local-first — data lives in a folder on your machine and works fully offline;
  • Data folder — view / change it under Settings → System (the "vault");
  • Auto-backup — a rolling backup is made before every save; restore in one click under Settings → System → Backup.

Auto-update

On launch the app quietly checks for and downloads updates in the background, with a small progress widget in the bottom-left; when it's ready, click Restart to update (or it installs automatically on next quit). You can also check manually under Settings → About. Installers are signed + notarized.

FAQ

Can I choose the install path on Windows?

Yes. The installer has a "Choose install location" page; it installs per-user by default (no admin needed) and can go on any drive.

macOS says "cannot verify the developer"?

Official builds are signed + notarized, so normally you won't see this. If you do, right-click → Open.

Is my data uploaded to the cloud?

Not by default. Only after you explicitly enable sync is data written to your chosen private GitHub repo / WebDAV; keys always stay on your machine.

Which platforms / architectures are supported?

macOS (Intel + Apple Silicon), Windows (x64 / ARM64 / x86), Linux (AppImage).
